## 一、项目背景与目标

### 1\.1 项目背景

现代年轻人（22\-35 岁职场新人、学生、自由职业者）日常消费场景频繁，但缺乏随手记账的轻量工具。现有记账 App 功能臃肿、操作路径长，导致用户难以坚持记录。市面上缺少一款 **移动端优先、操作极简、数据云端隔离** 的 Web 记账应用。

### 1\.2 产品定位

|项目|内容|
|---|---|
|产品名称|个人记账本|
|产品定位|轻量级、移动端优先的个人记账 Web 应用，支持多用户注册登录，数据云端隔离|
|目标用户|22\-35 岁年轻人群（职场新人、学生、自由职业者）|
|核心价值|快速记录 → 自动归类 → 可视化洞察 → 消费预警|
|使用场景|日常消费后随手记录；月底复盘消费结构；设置预算控制开支|

### 1\.3 产品目标

1. 提供极简的记账录入体验，单笔记录操作不超过 3 步

2. 开箱即用，首次登录即拥有预设分类和基础报表

3. 通过可视化图表帮助用户理解消费结构，培养理财意识

## 二、用户角色

|角色|描述|核心诉求|
|---|---|---|
|**未注册访客**|尚未注册账号的用户|快速完成注册，开始记账|
|**已注册用户**|已完成注册的普通用户|记录收支、管理分类、查看报表|
|**注销用户**|主动注销账户的用户|彻底删除个人所有数据|

> 暂不设管理员角色。

## 六、UI 原型 / 布局图

### 6\.1 设计原则

- **移动端优先**：所有页面以手机屏幕（375px\~414px 宽度）为设计基准

- **极简风格**：减少不必要的视觉元素，聚焦核心操作

- **底部导航栏**：主要页面通过底部 Tab 切换

### 6\.2 页面结构

```Plaintext
┌─────────────────────────┐
│         状态栏           │
├─────────────────────────┤
│     页面标题 / 头部       │
├─────────────────────────┤
│                         │
│                         │
│      核心内容区           │
│                         │
│                         │
├─────────────────────────┤
│  ───── 悬浮"+"按钮 ─────  │
├─────────────────────────┤
│  首页  │  账单  │  我的   │  ← 底部 Tab 导航
└─────────────────────────┘
```

### 6\.3 页面布局说明

#### 6\.3\.1 首页（月度概览）

- **顶部**：显示当前月份（支持左右滑动切换）及年份

- **中部卡片**：收入合计（绿色）、支出合计（红色）、结余（蓝色），三组数字横向排列

- **饼图区域**：分类支出占比饼图，点击图例可筛选

- **底部 Tab**：首页、账单、我的

#### 6\.3\.2 账单列表页

- **列表项**：分类图标 \+ 分类名称 \+ 备注（若有） \| 金额（支出红色/收入绿色）

- **交互**：左滑显示红色"删除"按钮；长按进入编辑模式

- **底部**：悬浮"\+"按钮，点击进入新增账单页

#### 6\.3\.3 新增账单页（从底部弹出 Modal）

- **金额输入**：数字键盘，默认聚焦，显示 ¥ 前缀

- **分类选择**：图标网格展示（预设 \+ 自定义），选中高亮

- **备注输入**：文本框，选填

- **日期**：默认当天，可点击修改

- **保存按钮**：置底

#### 6\.3\.4 登录/注册页

- **顶部**：应用 Logo 和名称

- **Tab 切换**：登录 \| 注册

- **表单**：邮箱输入框、验证码输入框（含"获取验证码"按钮）、密码输入框（注册时必填）

- **提交按钮**：全宽按钮

#### 6\.3\.5 我的页面

- **头像/用户信息**：邮箱显示

- **功能列表**：分类管理、设置、注销账户、退出登录

## 八、边界条件

### 8\.1 输入边界

|场景|处理策略|
|---|---|
|金额输入为 0 或负数|前端校验，提示"金额必须大于 0"，禁止保存|
|金额超过 999,999,999\.99|前端限制最多 11 位整数 \+ 2 位小数|
|备注超过 200 字|前端截断，仅保存前 200 字|
|邮箱格式不合法|前端实时校验，提示"请输入正确的邮箱地址"|
|验证码输入非数字字符|输入框限制仅数字输入，最大 6 位|
|密码长度不足（\< 6 位）|前端提示"密码至少 6 位"|
|自定义分类名称为空或超过 10 字|禁止保存，提示"分类名称 1\-10 个字"|

### 8\.2 业务边界

|场景|处理策略|
|---|---|
|删除已被账单使用的分类|提示"该分类下有 X 笔账单，删除后账单将归入'未分类'"|
|删除最后一条账单后查看首页|首页显示收入 ¥0、支出 ¥0、结余 ¥0，饼图显示"暂无数据"|
|某月无任何账单|首页显示收入 ¥0、支出 ¥0、结余 ¥0，饼图显示"暂无数据"|
|连续快速点击"获取验证码"|设置 60 秒倒计时冷却，防止频繁发送|
|验证码过期（超过 5 分钟）|提示"验证码已过期，请重新获取"|
|注销账户后使用同一邮箱重新注册|视为全新账号，无历史数据残留|

### 8\.3 并发与安全边界

|场景|处理策略|
|---|---|
|Token 过期|自动跳转登录页，提示"登录已过期，请重新登录"|
|多设备同时登录同一账号|后登录设备踢出前设备，Token 失效|
|请求未携带有效 Token|后端统一返回 401，前端跳转登录页|

## 九、非功能性需求

### 9\.1 性能要求

|指标|要求|
|---|---|
|页面首次加载时间|≤ 2 秒（正常 4G 网络）|
|账单列表加载|≤ 1 秒（1000 条以内）|
|新增/编辑/删除操作响应|≤ 500ms|
|并发用户数|支持 100 人同时在线|

### 9\.2 可用性要求

- 系统可用时间 ≥ 99\.5%（月度统计）

- 支持主流浏览器最新版本（Chrome、Safari、Edge、Firefox）

- 移动端适配：iOS Safari、Android Chrome

### 9\.3 安全要求

|项目|要求|
|---|---|
|密码存储|使用 bcrypt 或 argon2 哈希加密，禁止明文存储|
|数据传输|全站 HTTPS，敏感接口使用请求签名|
|身份认证|使用 JWT Token，有效期 7 天，支持刷新|
|验证码|6 位数字，有效期 5 分钟，单次有效|
|数据隔离|用户数据严格按 user\_id 隔离，禁止越权访问|
|注销安全|注销操作需二次确认，物理删除所有用户数据|

### 9\.4 可维护性

- 前端使用组件化开发，账单列表、分类选择器等核心组件可复用

- 后端 API 遵循 RESTful 设计规范，统一返回格式 `{ code, data, message }`

- 数据库表设计附带迁移脚本，支持版本化管理

### 9\.5 技术栈建议

|层级|技术选型|
|---|---|
|前端框架|Vue 3 / React|
|UI 组件库|Vant（移动端优先）|
|图表库|ECharts（饼图等可视化）|
|后端框架|Node\.js \(Express/Nest\) / Python \(FastAPI\)|
|数据库|PostgreSQL / MySQL|
|缓存|Redis（验证码、Token 黑名单）|
|部署|Docker \+ Nginx 反向代理|
